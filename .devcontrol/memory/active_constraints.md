# SP-DevControl - Restricciones Activas

- No tocar codigo fuente sin objetivo claro y trazable.
- Toda tarea relevante debe convertirse en checklist de sesion.
- No eliminar archivos sin aprobacion explicita.
- No sobrescribir documentos existentes sin control.
- Mantener naming oficial: SP-DevControl.
- Mantener naming tecnico local: .devcontrol y devcontrol.config.json.
- Tratar DevSentinel solo como antecedente del prototipo.
- Priorizar arquitectura Tauri + Rust como destino final.
