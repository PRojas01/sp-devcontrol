# Sesiones de SP-DevControl

## Estructura
- Una carpeta por fecha: `YYYY-MM-DD/`
- Dentro de cada fecha:
- `session-001.md`
- `session-001-checklist.md`
- `session-001-context.json`
- `session-001-diff-summary.md`
- `session-001-audit.jsonl`

## Regla operativa
- Toda solicitud relevante del usuario debe convertirse en checklist verificable.
- No cerrar una sesion sin estado final y pendientes claros.
